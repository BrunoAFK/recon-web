# v1.1.0
Open-core architecture — extensible server, component exports, handler fixes.

## Highlights

- **Pluggable server** — `buildServer()` now accepts `plugins` and `routes` options, enabling external repositories to inject middleware, auth, or
custom routes without modifying core code.
- **Component library exports** — both API and web packages export their internal modules (`exports.ts`), allowing external frontends to import
components, hooks, and API utilities.
- **Auth removed from open-source** — authentication, admin panel, and user management extracted to a separate proprietary repository. The
open-source version runs fully without auth — all routes public, no login UI.

## Bug Fixes

- **PageSpeed handler** — fixed "invalid argument" error by adding `https://` prefix when URL lacks a protocol and replacing deprecated `PWA` and
`BEST_PRACTICES` categories with `BEST-PRACTICES`.
- **PageSpeed error reporting** — API errors now include status code, message, and reason from Google's response instead of a generic "Request
failed with status code 400".
- **DELETE requests** — fixed `FST_ERR_CTP_EMPTY_JSON_BODY` error caused by sending `Content-Type: application/json` header on requests without a
body.
- **CSS @import order** — moved Google Fonts import before Tailwind to fix the "@import must precede all rules" warning.

## New Features

### BuiltWith Renderer Redesign

The BuiltWith Features renderer now shows a concise summary with technology group chips (active highlighted) and a "View all technologies" button
that opens a full-screen modal with:
- Stats bar (total, active, past, categories)
- Active/All toggle to filter inactive technologies
- Two-column grid of technology groups with category chips

### PageSpeed Quality Renderer

Scores now display one category per row (label left, score right) instead of a cramped horizontal grid.

### Nav Extensibility

Nav component accepts an optional `userMenu` slot prop, allowing external packages to inject custom dropdown menus (e.g. auth UI) without forking
the component.

## Breaking Changes

- `buildServer()` signature changed — now accepts optional `BuildServerOptions` parameter. Existing calls without arguments continue to work.
- `initDb()` signature changed — now accepts optional `DbMigration[]` parameter for schema extensions.
- Auth API endpoints (`/api/auth/*`, `/api/admin/*`) no longer exist. Consumers relying on these must use the proprietary extension.
- `@recon-web/api` package now has an `exports` field — import from `@recon-web/api` resolves to `exports.ts`, not `index.ts`.

## Upgrade Notes

- Delete your SQLite database (`data/recon-web.db`) before upgrading — the schema has changed (user tables removed, `user_id` column added to
scans).
- Remove auth-related environment variables from `.env` (`AUTH_PROVIDER`, `JWT_SECRET`, `SETUP_TOKEN`, etc.) — they are ignored.
- The `create-admin` CLI command has been removed.
