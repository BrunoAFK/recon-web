# v1.3.0

Bug-fix release. Fixes safeFetch failures on Node 24+, broken map tiles,
and missing external resources blocked by CSP.

## Bug fixes

- **safeFetch broken on Node 24+** — The custom DNS-pinning agent in
  `safeFetch` used the legacy `cb(null, address, family)` callback style.
  Node 24+ sends `{ all: true }` in lookup options and expects an array
  response `[{ address, family }]`. Every handler that uses `safeFetch`
  (firewall, headers, cookies, social-tags, robots.txt, linked-pages,
  tech-stack, rank, threats, quality, archives, http-security) was
  returning `"Invalid IP address: undefined"`. Fixed by respecting the
  `all` flag in the pinned lookup callback.
- **Server Location map off-center** — Leaflet miscalculated the map
  center when initialized in a container that hadn't finished layout.
  Added `invalidateSize()` + `setView()` after initialization to force
  correct centering. Zoom increased from 2 to 4 for better visibility.
- **CSP blocking Google Fonts, map tiles, and OG images** — The strict
  `Content-Security-Policy` header was missing allowances for
  `fonts.googleapis.com` (style-src), `fonts.gstatic.com` (font-src),
  `basemaps.cartocdn.com` map tiles, and external OG preview images
  (img-src). Updated to allow these external resources.

## Hardening

- **URL normalization at scan entry points** — `executeScan()` (API) and
  `runHandlers()` (CLI) now normalize the URL once before passing it to
  handlers, ensuring all handlers receive a fully qualified URL with
  protocol prefix. Screenshot handler also adds its own scheme validation
  as defense-in-depth.
