# v0.6.0

> **Early development release.** This project is still a work in progress. Expect breaking changes, incomplete features, and rough edges. Not recommended for production use yet.

First public release of recon-web - an OSINT website analysis tool with 33 security, DNS, network, and content checks.

## What's changed

### Core
- 33 analysis handlers covering DNS, SSL/TLS, WHOIS (100+ TLDs), ports, tech stack, security headers, and more
- Handler registry with metadata, categories, and runtime capability declarations
- Concurrent execution with configurable concurrency (p-limit)
- Error codes and categories (tool/site/info) for structured error handling
- SSRF protection - blocks private/internal IP addresses before handler execution
- URL scheme whitelist - rejects non-HTTP schemes (ftp://, file://, etc.)
- Hostname validation for traceroute (defence-in-depth against command injection)

### API
- Fastify REST server with Swagger documentation at /docs
- Real-time scan progress via Server-Sent Events (SSE)
- Scan history with SQLite persistence (WAL mode)
- HTML and PDF report generation with issue extraction and severity classification
- Centralized URL validation middleware (returns 400 before handlers run)
- In-flight scan deduplication (concurrent requests for same URL share one scan)
- Optional bearer token authentication with timing-safe comparison
- Scheduled scans with cron expressions and change detection
- Telegram and email notifications for detected changes
- Rate limiting (100 requests per 10 minutes)

### Web UI
- React + Vite + TailwindCSS frontend with real-time streaming results
- Category filters (Security, DNS, Network, Content, Meta, Performance)
- Status filters (All, OK, Issues, Info, Skipped)
- 36 specialized result renderers
- Scan history with infinite scroll pagination
- Side-by-side scan comparison with visual diff
- PDF/HTML report download
- Dark/light theme support
- Optional login page for authenticated instances

### CLI
- 33 auto-generated sub-commands from handler registry
- Output formats: text (coloured), JSON, JUnit XML
- Threshold gates for CI/CD (e.g. --fail-on ssl:expired)
- Diff mode for comparing with previous scans
- Handler filtering with --only flag

### Infrastructure
- Docker multi-stage builds for api, web, and cli
- All base images pinned to digest SHA for supply chain security
- GitHub Actions and GitLab CI pipelines with identical logic
- Trivy security scanning (CRITICAL for images, CRITICAL+HIGH for code)
- Docker images only published on release tags
- Prerelease tags (alpha/beta/rc) don't overwrite :latest
- Cloudflare Pages edge deployment (~16 HTTP-only handlers)
- Kubernetes Helm chart
- Release script (./scripts/version) for automated versioning
