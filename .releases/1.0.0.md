# v1.0.0

The first stable release of recon-web — 39 checks, 3 interfaces, fully self-hosted.

## Highlights

- **Self-hosted SSL grading** — no more dependency on SSL Labs API. Grades A+ to F computed locally using Node.js TLS, checking protocol version, cipher strength, certificate validity, forward secrecy, and HSTS. Results are instant (~1s).
- **Documentation site** — 24-page Astro Starlight docs site with full-text search (Pagefind), dark/light theme, and detailed explanations of all 39 checks including what a good result looks like.
- **Rewritten handlers** — sitemap, TLS, TXT records, and SSL grade handlers rebuilt for reliability.

## Bug Fixes

- **Scan completion page refresh** — live scan results now update in-place when the scan completes instead of triggering a full page reload.
- **Sort reading order** — replaced CSS columns with a DOM-aware JS masonry grid that keeps correct left-to-right sort order while maintaining tight vertical packing.
- **Sitemap detection** — handler now checks `robots.txt` first for sitemap directives, then falls back to common paths (`/sitemap.xml`, `/sitemap_index.xml`). Handles case-insensitive XML tags.
- **TXT records** — added root domain fallback when subdomain has no TXT records (e.g., `www.example.com` → `example.com`). Fixed duplicate key handling and renderer.
- **TXT records renderer** — now supports key-value format returned by the handler, previously only recognized array format.
- **Search input** — removed duplicate X button caused by `type="search"` native clear button overlapping custom clear button.
- **SQLite test isolation** — each API test file uses a unique temporary database to prevent `SQLITE_BUSY` errors in parallel CI runs.

## New Features

### Self-Hosted SSL Grade (replaces SSL Labs API)

The `ssl-labs` handler no longer calls the Qualys SSL Labs API. Instead, it connects directly to the target using Node.js `tls` module and computes a grade based on:

| Check | A+ | A | B | C/D | F |
|-------|-----|-----|-----|------|-----|
| Protocol | TLS 1.3 | TLS 1.2 | — | TLS 1.0/1.1 | SSL/no TLS |
| Cipher | AEAD | AEAD | CBC | — | RC4/DES/NULL |
| Certificate | Trusted, >30d | Trusted | Trusted, <30d | — | Expired/untrusted |
| Forward secrecy | ECDHE/DHE | ECDHE/DHE | — | None | — |
| HSTS | ≥1 year | Present | — | Missing | — |

The renderer shows a detailed checklist of each check with pass/fail status.

### Native TLS Handler

Replaced the dead Mozilla TLS Observatory API (`tls-observatory.services.mozilla.com`) with a native Node.js `tls` connection that returns protocol version, cipher suite, and certificate details directly.

### Rewritten Sitemap Handler

- Checks `robots.txt` first for `Sitemap:` directives (supports multiple entries)
- Falls back to `/sitemap.xml` and `/sitemap_index.xml`
- Uses `normalizeTags: true` for case-insensitive XML parsing
- Recursively extracts all `<loc>` values regardless of nesting
- Returns structured data: `{ sitemapUrl, source, type, urls }`

## Upgrade Notes

- The `ssl-labs` handler no longer requires network access to `api.ssllabs.com`. If you had firewall rules allowing that endpoint, they can be removed.
- The `tls` handler no longer requires network access to `tls-observatory.services.mozilla.com`.
- Sitemap handler returns a new data structure (`{ sitemapUrl, source, type, urls }`) instead of raw xml2js output. If you consume the API programmatically, update your parsing.
- TXT records handler returns a key-value object instead of the previous format. Keys without `=` are stored as-is. Duplicate keys get `(2)`, `(3)` suffixes.
