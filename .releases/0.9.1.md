# v0.9.1

Patch release — fixes failing CI tests and adds automated release creation.

## Bug Fixes

- **Registry test** — handler count is now dynamic instead of hardcoded (was failing after adding new handlers)
- **Scan test** — URL reachability pre-check is skipped when running targeted handler subsets (was failing in CI where `example.com` is unreachable)

## CI/CD

- **Automated GitHub Release** — CI now creates a GitHub Release after all checks pass; uses `.releases/<version>.md` if present, otherwise creates a draft with auto-generated notes
- **Automated GitLab Release** — same behavior for GitLab CI pipelines
- No more releases with failing tests — release job depends on lint, test, audit, and docker stages

---

*This release includes all changes from v0.9.0:*

## New Handlers

- **Server Location** — Geolocate server by IP with interactive Leaflet map (ip-api.com, no key required)
- **SSL Labs Grade** — Qualys SSL Labs A+/A/B/C/F grading with endpoint details (no key required)
- **SEO Audit** — On-page SEO score (0-100): title, meta description, headings, images, structured data, word count
- **WordPress Scan** — Detect WP installations, enumerate plugins/themes/versions, check exposed files (xmlrpc.php, debug.log, etc.)
- **VirusTotal** — Scan URL against 70+ antivirus engines (free API key: 500 req/day)
- **AbuseIPDB** — IP reputation and abuse confidence score (free API key: 1,000 req/day)

## UI Redesign

- **Fullscreen scanning view** — animated progress screen during scan, "Show results now" button after 8s
- **Scan URL persistence** — browser URL updates to `/history/:scanId` immediately, refresh loads from history
- **Filter bar redesign** — dropdown menus for Category, Status, Group, and Sort
- **Grouping** — group results by category or status with collapsible sections (Collapse all / Expand all)
- **Sorting** — A→Z / Z→A name sorting, OK first / Issues first status ordering
- **Verdict component** — prominent pass/fail display for HSTS, Firewall, security.txt, WordPress, DNSSEC
- **Stacked layouts** — Social Metadata, HTTP Headers now use label-above-value format for better readability
- **Card description width** — summary text now uses full card width instead of sharing space with action buttons
- **Stale scan timeout** — 15s client-side timeout for unresponsive scans

## Renderer Improvements

- **Traceroute** — visual hop chain with arrows, collapsed `*` gaps, TCP fallback for Docker environments
- **DNSSEC** — shows DNS response flags (RD, RA, TC, AD, CD) for all record types, always displays results
- **DNS Records** — cleaner section-based layout, A record inline, AAAA/MX/NS as sections
- **Redirects** — visual chain with ↳ icons and redirect count summary
- **Headers** — stacked name/value format with show more toggle

## Mobile & Responsive

- **History results** — header buttons and metadata wrap properly on mobile
- **History list** — scan entries use flex-wrap for metadata
- **Compare page** — stacked mobile layout for diff tables, responsive scan headers
- **Stat badges** — improved vertical wrapping

## Other Improvements

- **URL reachability pre-check** — DNS lookup first (fast fail ~1-3s for non-existent domains), then HTTP HEAD
- **Carbon handler** — fixed URL normalization (works without `https://` prefix)
- **CLI exit fix** — `process.exit(0)` ensures clean exit after scan
- **CLI config** — added VIRUSTOTAL_API_KEY and ABUSEIPDB_API_KEY env var support
- **GitHub link** — corrected to `github.com/BrunoAFK/recon-web` across all pages
- **Footer redesign** — 4-column layout with brand, product links, resources, CLI command
- **Homepage** — dynamic handler count, CLI oneliner with copy button, handler list shows displayName
- **Settings** — display preferences (grouping, sorting, status order), full API key docs with signup links and free tier limits, NOTIFY_METHOD env var, complete SMTP config
- **PDF Report** — button text clarified to "PDF Report"
- **Ad placeholder removed** from homepage

## Settings & Configuration

New environment variables:
- `VIRUSTOTAL_API_KEY` — VirusTotal integration (free: 500/day)
- `ABUSEIPDB_API_KEY` — AbuseIPDB integration (free: 1,000/day)
- `NOTIFY_METHOD` — notification delivery: `telegram`, `email`, or `both` (default)

Client-side preferences (localStorage):
- Default grouping (none / category / status)
- Default sorting (A→Z / Z→A)
- Status group order (OK first / Issues first)
